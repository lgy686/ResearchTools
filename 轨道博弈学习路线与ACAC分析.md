# 轨道博弈学习路线与 ACAC 算法适配分析

## 1. 你的当前起点与核心目标

你目前的状态可以概括为：

- 你是研一硕士，现在处于研一下学期。
- 研究方向是轨道博弈。
- 你此前在该方向上几乎没有基础。
- 你目前已经会下载卫星轨道 TLE 数据。
- 你希望后续能顺利毕业，并形成论文成果。

在这种情况下，最重要的不是立刻去追最复杂、最前沿的算法，而是尽快建立一条可以真正产出论文的最小研究闭环：

`轨道数据 -> 轨道动力学建模 -> 博弈问题定义 -> 基线算法 -> 改进算法 -> 实验评估 -> 论文写作`

你现在最大的问题不是“缺一个算法”，而是“还没有把轨道数据、动力学模型、博弈建模和智能决策方法连成一个完整链条”。


## 2. 对 ACAC 的直接结论

你给出的论文和仓库分别是：

- 论文：[OpenReview 论文页面](https://openreview.net/forum?id=323GZNnGqe)
- 论文 PDF：[OpenReview PDF](https://openreview.net/pdf?id=323GZNnGqe)
- 仓库：[ACAC GitHub 仓库](https://github.com/LGAI-Research/acac?tab=readme-ov-file)

### 2.1 ACAC 是什么

ACAC 不是一个“专门用于轨道博弈”的算法，而是一个用于**异步多智能体强化学习**的问题框架与方法。它主要解决的是：

- 多个智能体不是同步决策的；
- 智能体执行的是持续时间不同的宏动作（macro-actions）；
- 在这种异步条件下，如何更稳定地做集中训练、分散执行；
- 如何构造适合异步场景的 critic、历史表示与 advantage 估计。

简单理解，ACAC 更适合这种问题：

- 每个智能体不是每个离散时刻都行动一次；
- 一个动作执行后可能持续几秒、几十秒、几分钟甚至更久；
- 不同智能体完成动作的时间不同，所以新的决策时刻也不同。

### 2.2 它能不能用于轨道博弈

结论是：**可以，但不能直接拿来即用，必须先完成轨道问题建模与环境改造。**

它适合你的前提包括：

- 问题中有多个智能体，例如多个追击星或多个防御星；
- 动作可以抽象成高层的轨道机动或策略动作，而不是每一时刻输出连续推力；
- 不同动作持续时间不同，天然存在异步决策；
- 研究重点是协同围捕、协同追踪、协同防御，或者先固定对手策略再训练本方策略。

它不太适合直接原样使用的情况包括：

- 你的问题核心是连续低推力控制；
- 你的研究目标是严格求双边零和博弈均衡；
- 你想把原始 TLE 字符串直接输入算法并让它自动学会轨道博弈。

### 2.3 为什么说它“适合，但不能直接用”

原因有三点：

- ACAC 论文和仓库本身解决的是通用异步多智能体决策问题，不是专门的航天轨道问题。
- 仓库中的现成环境主要是 benchmark 环境，不是轨道动力学环境。
- 轨道博弈首先需要你构建一个合理的轨道动力学仿真环境，算法只能建立在这个环境之上。

所以正确的顺序不是：

`TLE -> ACAC`

而是：

`TLE/OMM -> 轨道传播 -> 状态表示 -> 博弈环境 -> 奖励与约束 -> 基线算法 -> ACAC 迁移`


## 3. TLE 到底应该怎样使用

### 3.1 TLE 不是最终研究对象，而是场景入口

TLE 是 Two-Line Element，主要包含：

- 历元；
- 倾角；
- 升交点赤经；
- 偏心率；
- 近地点幅角；
- 平近点角；
- 平均运动等信息。

相关资料：

- [CelesTrak TLE 格式说明](https://celestrak.org/NORAD/documentation/tle-fmt.php)
- [CelesTrak 轨道文档](https://celestrak.org/NORAD/documentation/)

你要明确一点：

- TLE 不是一条“已经展开好的轨迹序列”；
- TLE 更像是某一历元附近的轨道描述；
- 它通常要配合 SGP4/GP 模型去传播，才能得到未来某时刻的位置速度。

#### 常见疑问1：TLE 到底表示“某一秒”还是“某一时间段”？

TLE 既不是“某一秒的精确状态”，也不是“严格定义的某个时间段轨迹”。

更准确地说：

1. TLE 是一个在历元（epoch）时刻定义的轨道参数集。
2. 这些参数是用一段观测数据拟合出来的，所以它本质上代表“历元附近最可信”的轨道描述。
3. 需要用 SGP4 从这个历元向前或向后传播，得到任意时刻的位置速度；离历元越远，误差通常越大。

一句话记忆：`TLE = 历元附近的拟合模型参数，不是单秒真值，也不是完整时间段轨迹`。

#### 常见疑问2：什么是历元（epoch）？

历元就是这组轨道参数“生效/参考”的基准时刻。

对 TLE 来说：

1. TLE 里的各参数（如平近点角、RAAN 等）是指在这个历元时刻的状态。
2. 使用 SGP4 传播，本质是“从历元出发”推到别的时刻。
3. 离历元越远，通常误差越大。
#### 常见疑问3：什么是用 SGP4 将 TLE 传播到统一历元？

`先选一个共同时间点 t0，然后把每颗卫星的 TLE 都用 SGP4 推算到 t0 的状态（r,v）。`

不是把 TLE 文件“改成同一个历元”，而是把状态向量对齐到同一时刻。

例子：

1. 卫星 A 的 TLE 历元：2026-03-27 00:10:00 UTC
2. 卫星 B 的 TLE 历元：2026-03-26 23:40:00 UTC
3. 你选统一历元：t0 = 2026-03-27 01:00:00 UTC
4. 用 SGP4 分别传播：
* A：从 00:10 推到 01:00
* B：从 23:40 推到 01:00
5. 得到 A(t0)、B(t0) 的 r,v 后，才能正确算相对位置/速度、再转 RTN/LVLH。
一句话：统一历元就是“把不同起点的轨道信息，拉到同一时间线上再比较”。

### 3.2 你不能直接把 TLE 当成强化学习输入

TLE 的正确用途通常是：

1. 从公开数据库中获取卫星轨道初始信息；
2. 用 SGP4 将 TLE 传播到统一历元；
3. 得到某一时刻的轨道状态向量 `r, v`；
4. 再把状态转成适合博弈建模的形式。

### 3.3 真正适合博弈建模的状态表示

后续研究中，你更常使用的不是 TLE 字符串，而是以下这些状态表示：

- 笛卡尔状态：位置和速度 `r, v`
- 轨道根数：`a, e, i, Ω, ω, M` 或真近点角
- 相对位置速度：在 RTN / RSW / LVLH 坐标系下的相对状态
- 相对轨道根数：ROE 或其他相对轨道参数

如果你的方向是轨道追逃、围捕、协同占位，那么最重要的桥梁知识是：

- `ECI -> RTN / LVLH`
- `绝对轨道 -> 相对轨道`
- `两体模型 / J2 摄动 / HCW(CW) 相对运动模型`


## 4. ACAC 在轨道博弈中的合理使用方式

### 4.1 最适合的题目类型

ACAC 最适合迁移到如下类型的问题：

`多追击者 - 单逃逸者的异步脉冲轨道博弈`

这个方向的优点是：

- 每次机动可以设计成宏动作；
- 机动后通常要经历漂移或等待，因此天然异步；
- 多个追击者之间需要协同，符合多智能体设定；
- 可以先把逃逸方做成规则策略，降低初期难度；
- 既有轨道动力学意义，又有强化学习创新空间。

### 4.2 如何把轨道机动映射成宏动作

你可以把轨道上的高层决策设计成离散的宏动作，例如：

- `coast for Δt`
- 沿切向正向脉冲
- 沿切向反向脉冲
- 沿径向正向脉冲
- 沿径向反向脉冲
- 沿法向正向脉冲
- 沿法向反向脉冲
- 调整相位
- 保持安全距离
- 进入观测窗口
- 转入拦截轨道

这里最重要的不是动作名字，而是两个原则：

- 动作是高层的、可解释的，不是每一步输出一个连续推力；
- 不同动作持续时间不同，这样 ACAC 的异步优势才能体现出来。

### 4.3 观测空间怎么设计

你可以把单个智能体的观测设计为：

- 本星相对目标的 RTN 相对位置和速度；
- 本星当前轨道根数或相对轨道量；
- 剩余燃料或累计 `Δv`；
- 与友方星之间的相对距离；
- 目标的估计状态；
- 上一个执行的宏动作；
- 距离上一次决策已经过去的时间；
- 是否处于可见窗口、通信窗口或安全约束边界附近。

### 4.4 奖励函数怎么设计

最开始不要过度复杂化，先保证“可解释、可调试”。

追击方的奖励可以包含：

- 距离目标更近时给正奖励；
- 进入捕获窗口时给较大正奖励；
- 多星包围构型更优时给协同奖励；
- 燃料消耗给惩罚；
- 碰撞风险或违反安全距离给大惩罚；
- 超时未完成任务给惩罚。

逃逸方的奖励可以包含：

- 存活时间越长奖励越高；
- 与追击方拉开距离给正奖励；
- 诱导对方耗费更多燃料给奖励；
- 被捕获或进入不利区域给惩罚。

### 4.5 一个重要提醒

ACAC 是多智能体强化学习方法，不等于已经解决“双边博弈均衡”问题。

如果你后续做的是严格零和或双边对抗场景，那么还可能需要额外加入：

- self-play；
- 交替训练；
- 多对手池；
- PSRO 或其他近似博弈求解机制。

所以更稳妥的研究路径通常是：

- 先固定逃逸者策略，训练多追击者协同策略；
- 再考虑双边共同学习；
- 最后再讨论博弈均衡或更强对抗性。


## 5. 你应该准备哪些数据

强化学习研究中，最重要的并不是“带标签的数据集”，而是“环境参数、场景分布和评测场景集”。

你至少要准备以下五类内容。

### 5.1 轨道初始条件

- 目标星、追击星、协同星的 TLE 或 OMM 数据；
- 传播到统一历元后的位置速度状态；
- 典型轨道类型，例如：
  - 近圆 LEO；
  - 近共面不同相位；
  - 不同高度差；
  - 不同倾角差；
  - 不同初始相对距离。

### 5.2 动力学与任务约束参数

- 地球引力常数 `μ`；
- 地球半径；
- 是否考虑 J2 摄动；
- 是否考虑大气阻力；
- 单次最大脉冲 `Δv`；
- 总燃料预算；
- 最小安全距离；
- 仿真时长上限；
- 机动冷却时间或动作持续时间。

### 5.3 感知与通信参数

- 是否能够观测对手真值状态；
- 是否存在估计误差；
- 观测噪声大小；
- 通信是否全连接；
- 信息更新周期；
- 可见窗口或地球遮挡约束。

### 5.4 博弈规则

- 什么条件算成功捕获；
- 什么条件算成功逃逸；
- 是否允许碰撞风险；
- 是否有禁飞区域；
- 是否要求保持编队安全距离；
- 是否是单边训练还是双边对抗。

### 5.5 评测场景集

你需要把实验场景分为：

- 训练场景；
- 验证场景；
- 测试场景；
- 未见初始轨道几何场景；
- 未见对手策略场景。

后面写论文时，你的评价指标最好包括：

- 捕获成功率；
- 平均捕获时间；
- 平均累计 `Δv`；
- 安全违规率；
- 对不同初始构型的泛化能力；
- 对不同对手策略的鲁棒性。


## 6. 建议的学习路线

下面这条路线是基于你当前基础、毕业目标和论文需求设计的，核心原则是“先搭稳底座，再做算法创新”。

### 第一阶段：现在到暑假前

目标：补齐轨道动力学和计算工具基础。

你需要学会：

- 两体轨道力学最基本概念；
- 轨道六根数与位置速度的关系；
- 常用坐标系：ECI、ECEF、RTN、LVLH；
- TLE 字段含义；
- SGP4 轨道传播；
- Python 科学计算基础：`numpy`、`scipy`、`matplotlib`。

这一阶段必须完成的最小成果：

- 能读入一组 TLE；
- 能传播一颗卫星未来 24 小时轨道；
- 能画出位置或轨道变化图；
- 能把两颗卫星传播到同一历元；
- 能计算它们的相对位置和相对速度；
- 能转换到 RTN 或 LVLH 坐标系。

如果这一步没有完成，后续的轨道博弈和强化学习会非常吃力。

### 第二阶段：暑假到研二上前半

目标：从“轨道传播”进入“轨道机动与相对运动建模”。

你需要学会：

- HCW/CW 相对运动方程；
- 相对轨道运动的基本分析方法；
- 脉冲机动的基本建模；
- 相对状态的传播；
- 任务约束如何转化为仿真规则。

这一阶段建议完成的成果：

- 一个最小的 2D 或 3D 追逃仿真器；
- 一个基于规则的追击策略 baseline；
- 能比较不同初始条件下的成功率、时间和耗能。

### 第三阶段：研二上前半

目标：正式把问题写成博弈环境。

你需要学会：

- MDP；
- POMDP；
- Dec-POMDP；
- MacDec-POMDP；
- 奖励函数设计；
- 终止条件与约束的程序化表达。

这一阶段应形成的成果：

- 一个清晰的问题定义文档；
- 完整写明状态、动作、奖励、终止条件；
- 实现一个可重复运行的轨道博弈环境。

### 第四阶段：研二上后半

目标：先做基线，不要直接上 ACAC。

你需要学习：

- DQN 或 PPO 的基础原理；
- MAPPO、MADDPG 或 CTDE 基本思想；
- centralized critic 与 decentralized actor 的作用。

建议顺序是：

1. 先做单智能体 PPO；
2. 再做双智能体或多智能体 baseline；
3. 再做同步决策版本；
4. 最后再迁移到 ACAC 这类异步多智能体框架。

这样做的原因是，你先要确认问题是否来自：

- 动力学建模；
- 环境设计；
- 奖励函数设计；
- 训练不稳定；
- 算法实现本身。

### 第五阶段：研二寒假到研二下前半

目标：把 ACAC 迁移到你的轨道博弈环境。

这一阶段你重点做四件事：

1. 定义轨道宏动作
2. 设计异步环境事件机制
3. 接入 ACAC 的历史编码与 critic 思路
4. 做消融实验和对比实验

你至少要比较：

- 是否使用时间嵌入；
- 是否使用 agent-centric critic；
- 是否采用更适合宏动作的 advantage 估计；
- 是否比同步版本更稳定、更高效。

### 第六阶段：研二下

目标：收敛实验、形成论文、准备毕业相关成果。

此时你的论文主线应当比较清晰：

- 为什么轨道博弈中存在异步宏动作；
- 为什么同步决策方法不够合适；
- 你如何构建轨道追逃环境；
- 你如何把 ACAC 迁移到该环境；
- 你的方法相对基线有哪些改进；
- 你的方法在哪些条件下有效、在哪些条件下失效。


## 7. 你现在最应该先做的四件事

如果你现在就想开始，不建议继续只看论文，而建议先做以下四件具体任务。

### 任务 1：完成 TLE 到状态向量的传播

目标：

- 用 Python 读入一组 TLE；
- 使用 SGP4 将其传播到未来 24 小时；
- 每隔固定时间输出位置和速度。

### 任务 2：完成双星相对状态计算

目标：

- 选择两颗卫星；
- 传播到同一历元；
- 计算两星的相对位置和相对速度；
- 将结果转换到 RTN 坐标系。

### 任务 3：写一个最简单的轨道追击环境

目标：

- 先不要追求真实度过高；
- 让追击者只能从有限个离散脉冲动作中选择；
- 环境能返回状态、奖励、是否结束。

### 任务 4：先做基线，再考虑 ACAC

目标：

- 先写一个规则追击器；
- 再训练一个最简单的 PPO baseline；
- 先验证环境和奖励是否合理；
- 最后再考虑把 ACAC 迁移进来。


## 8. 推荐的研究题目方向

如果你的目标是“稳妥毕业，并尽可能形成论文成果”，我比较推荐你把问题收敛到下面这类题目：

`基于异步宏动作强化学习的多航天器脉冲轨道追逃博弈`

这个方向的优点有：

- 和 ACAC 的问题设定天然匹配；
- 研究主线明确；
- 工程实现复杂度相对可控；
- 有较清晰的轨道动力学背景；
- 容易形成“建模 + 算法 + 实验”的完整论文结构；
- 既可以做协同，也可以逐步扩展到对抗。

如果你的导师更偏传统航天控制、优化或者博弈论，那么在论文表述上建议采用：

- 轨道追逃/围捕建模是主体；
- 强化学习只是求解方法之一；
- 同时给出规则法、经典 RL 法或传统方法作为对比。

这样更容易被接受，也更容易让工作显得“问题驱动”，而不是“为了套算法而套算法”。


## 9. 你目前不用过度焦虑的地方

你现在并不是“完全没有基础”，因为你已经掌握了一个重要入口：**能够获取轨道数据**。

这意味着你已经迈出了第一步。接下来要做的是把这一步逐渐扩展成完整能力链：

- 从 TLE 到轨道传播；
- 从轨道传播到相对运动；
- 从相对运动到博弈建模；
- 从博弈建模到算法求解；
- 从算法求解到论文实验。

不要试图在很短时间内同时彻底搞懂：

- 轨道力学；
- 博弈论；
- 多智能体强化学习；
- 论文写作。

更稳妥的方法是按模块推进，逐个击破。


## 10. 后续最建议继续推进的两个方向

你接下来最值得做的，不是继续泛泛看更多文章，而是从以下两个方向里选一个继续深入。

### 方向 A：做一个 12 周学习计划

适合你如果你现在最缺的是：

- 时间安排；
- 学习顺序；
- 每周到底该做什么；
- 如何避免学得很散。

这个计划可以细化到：

- 每周看什么知识；
- 每周写什么代码；
- 每周要形成什么成果；
- 每周如何验收。

### 方向 B：直接设计一个最小轨道博弈环境

适合你如果你现在更希望快速进入研究和实现阶段。

这个设计可以直接明确：

- 状态空间；
- 动作空间；
- 奖励函数；
- 终止条件；
- 轨道模型；
- 训练流程；
- 后续如何接 ACAC。

## 12. 一句话总结

ACAC 可以作为你未来研究中的一个重要算法候选，但它不是起点。你的真正起点应该是：

`学会把 TLE 变成轨道状态 -> 学会构建相对运动与追逃环境 -> 先做基线 -> 再迁移 ACAC`

只要你按这个顺序推进，即使目前基础较弱，也完全有机会在硕士阶段形成可以支撑毕业的论文工作。
